PHR 病人資訊標準化互通

包含的子系統: Patinet Portal、PHR FHIR server、DICOMWeb server、民眾端系統、醫護系統、數據分析端系統
Patinet Portal、PHR FHIR server、DICOMWeb server 由主辦單位提供。
聯測參與單位扮演民眾端系統、醫護系統、或數據分析系統腳色(client 端系統)。


民眾端系統

  1. 從 Portal 創建及取得帳號，並取得創建資料授權(2020 聯測先不做授權驗證)
  2. 在 PHR FHIR server、DICOMWeb server 創建病人資料(可完全匿名，
  3. 在 PHR FHIR server 上傳生理監測資料(測試項目二)、上傳用藥處方及用藥紀錄(測試項目三)
  4. 在 DICOMWeb server 上傳 DICOM 影像

醫護系統

  1. 民眾授權醫護系統調閱個人健康紀錄(授權驗證 2021 聯測)，民眾提供個人 PHR Patient.id 給醫護系統
  2. 醫護系統由 Patient.id 調閱生理監測資料(測試項目二)
  3. 醫護系統由 Patient.id 調閱用藥處方及用藥紀錄(測試項目三)
  4. 醫護系統由 Patient.id 調閱 imageingStudy(測試項目四)
  5. 醫護系統由 imageingStudy 向 DICOMWeb server 調閱 醫學影像
  6. 醫護系統由在影像檢視系統進行標記，並上傳標記到 DICOMWeb server (DICOM PS object) 或上傳標記到 FHIR server (SVG annotation)

數據分析系統

  1. 民眾簽署參與數據分析研究同意書，授權醫護系統調閱個人健康紀錄(授權驗證 2021 聯測)，民眾提供個人 PHR Patient.id 給數據分析系統
  2. 數據分析系統由 Patient.id 調閱生理監測資料(測試項目二)
  3. 數據分析系統由 Patient.id 調閱用藥處方及用藥紀錄(測試項目三)
  4. 擴充項目:數據分析系統基於病人生理監測及用藥等資訊產生及上傳分析結果(2020 不定分析結果規範，參與單位可參考 FHIR observation 上傳結果)
  5. 數據分析(影像 AI)系統由 Patient.id 調閱 imageingStudy(測試項目四)
  6. 數據分析(影像 AI)系統由 imageingStudy 向 DICOMWeb server 調閱 醫學影像
  7. 數據分析(影像 AI)系統由在影像檢視系統進行標記，並上傳標記到 DICOMWeb server (DICOM PS object) 或上傳標記到 FHIR server (SVG annotation)

授權憑證由 portal 核發給參與測試單位，不同情境及腳色，給與不同權限憑證。

MI-TW 2020 聯測不驗證權限憑證。但若有民眾實際授權提供健康紀錄，相關系統則須基於憑證，達成存取控制
